## 1. Линейный поиск (Linear Search)

Простейший алгоритм поиска, при котором элементы массива последовательно сравниваются с искомым значением (ключом), начиная с первого.

Принцип работы:
Алгоритм проходит по массиву слева направо и сравнивает каждый элемент с ключом. Поиск завершается при первом совпадении.

Сложность:

Худший случай: O(n)
Лучший случай: O(1)

Особенности:

Не требует сортировки массива
Прост в реализации
Неэффективен на больших данных
## 2. Бинарный поиск (Binary Search)

Эффективный алгоритм поиска в отсортированном массиве, основанный на делении диапазона пополам.

Принцип работы:
На каждом шаге сравнивается средний элемент массива с ключом:

если ключ меньше — поиск продолжается в левой части
если больше — в правой

Сложность:

O(log n)

Особенности:

Требует предварительной сортировки
Значительно быстрее линейного поиска на больших массивах
## 3. Интерполяционный поиск (Interpolation Search)

Улучшение бинарного поиска для равномерно распределённых данных.

Принцип работы:
Вместо середины массива выбирается позиция, вычисляемая на основе значения ключа (интерполяция).

Сложность:

Средний случай: O(log log n)
Худший случай: O(n)

Особенности:

Эффективен при равномерном распределении значений
Требует отсортированного массива
Может работать хуже бинарного при плохом распределении данных
## 4. Прямой поиск подстроки (Naive String Search)

Базовый алгоритм поиска подстроки в строке.

Принцип работы:
Подстрока сравнивается с каждой возможной позицией в строке посимвольно.

Сложность:

O(n * m) (n — длина строки, m — длина подстроки)

Особенности:

Простая реализация
Низкая эффективность при больших строках
## 5. Алгоритм Бойера — Мура (Boyer–Moore)

Один из самых эффективных алгоритмов поиска подстроки.

Принцип работы:
Сравнение начинается с конца шаблона. Используются эвристики:

"плохого символа"
"хорошего суффикса"

Это позволяет пропускать большие части строки.

Сложность:

Средний случай: O(n)
Худший случай: O(n * m)

Особенности:

Очень быстрый на практике
Сложнее в реализации
Эффективен для длинных строк
## 6. Алгоритм Бойера — Мура — Хорспула (Horspool)

Упрощённая версия алгоритма Бойера–Мура.

Принцип работы:
Использует только эвристику "плохого символа", что упрощает реализацию.

Сложность:

Средний случай: O(n)

Особенности:

Проще Бойера–Мура
Быстр в реальных задачах
Хороший баланс между сложностью и производительностью
